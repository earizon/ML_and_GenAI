## An Spreadsheet is all you need

* <https://github.com/dabochen/spreadsheet-is-all-you-need>

* A nanoGPT pipeline packed in a spreadsheet.

- I did to help myself understand how GPT works.
- Pretty fun to play with, especially when trying to figure out what
  exactly is going on inside a transformer.

- This helped me to visualize the entire structure and the data flow:
  - All the mechanisms, calculations, matrices inside are fully
    interactive and configurable.

... While reading about LLMs, I realised that the internal mechanisms of a 
transformer is basically a range of matrices calculations being connected in a 
certain order.

I started to wonder if the whole process can be represented in a spreadsheet 
since all the calculations are fairly simple.

It contains all the transformer components including:

- embedding
- layer norm
- self attention
- projection
- MLP
- softmax
- logits

It is based on Andrej Karpathy's NanoGPT structure which includes roughly 85000 parameters.

- WARN: this project is a character (vs token) based prediction system and to reduce
  the complexity, only letter A/B/C are being tokenized.

 In the numbers file "nanoGPT.numbers", you will see two essetially similar tabs:
- "no weights" tab    : parameters are very tidy and it shows weird values down the pipeline.
                        but it is also clearer to help you read it.
- "random weights" tab:

- Firstly, all the blocks are the values or parameters that is processed through
  the GPT architecture: 
  - Purple: parameters that are supposed be replaced by a trained model's parameter.
  - Green : the values that started from the input and gets transformed into the end results.
  - Orange: intermediate values used for the calculation.
