# Prompt "Engineering"

## Prompting 101


  It has been experimentally probed that Large Language Models (LLM) learn,
during the training phase, not only "lot of data and vocabulary" but also,
in their aim to "guess" the next word (next token actually), its internal
network of billions of parameters is also trained on human-language
"thinking process" allowing to replicate (or emulate) human reasoning and
even some reflection pattern ("evaluate my own answer from outside myself").
 The LLM is also able to capture succinct nuances like the language style
(formal, informal, direct, verbose, scientific, legal, computer language, ...)
and head answers and context according to them. They also also to be asked for
 
 LLM can also adapt to conditional questions ("If the user likes action movies,
recommend a recent release"). In this sense, prompt are "programmable" and emulate
loops and conditional branches as a software program. This is possible because
the underlying Neural Network with billions of parameters has captured this
possibility.
  TIP: be warned that it has captured it statistically, in a very
inefficient and unreliable way compared to traditional computer programming 
languages, still good enough for many purposes.

 LLMs They are not just "big" databases providing mechanic and simple 
question-answer flows as a convention search engine.  [[doc_has.keypoint]]

 To profit from the internal knowledge stored in the trained network, and
in direct contrast to big text databases (search engines), when consulting
to the LLM a first "MUST TO" is to **PROVIDE CONTEXT**. This context can be:

- The "world around" our problem or question and/or
- known related facts surrounding our prompt  and/or
- the expected way of thinking and/or
- The level of details or the style of the answer
- ...

To differentiate from conventional **search queries**, we speak of **PROMPTING**.

Providing context helps also to avoid hallucinations.
- Modern LLMs are trained using a GPT architecture, that basically try to find 
  the next most probable word based on the previous set of text. At some point the
  "most probable word" can fail or start deviating from the expected answer.
  The more context it has (the more "previous set of text" or the more previous hints)
  to guess the next word, the less probably it will be to hallucinate and take the
  wrong path.
- Since modern LLMs **do** understand human language we can use the prompt context 
  to also indicate what we **DO NOT** expect in the answer.


  Example prompt:
  ```
  You are a sociologist ellaborating a report that allows to guide social actions
  in the metropolitan area of Paris.
  - Detail data and about socio-demographic information and related trends.
  - Do NOT include information related to tourism in Paris.
  - Do NOT include any information related to the Olympic Games.
  - Exclude information older than 20 years.
  - Use and academic style for the answer.
  - Support your claims with any external reference to original sources
  - Highlight and emphasize any measurable metric.
  ```

**WARN**: In many scenarios (chatbots, LLM agents with long term evolution cycles) the prompt
needs frequent quality assurance.
- New LLM models are smarter and previous prompt can become outdated with non needed hints
  for the new models.
- Also the new LLM models can allow for new "thinking mode" prompts that previously failed.
  For example "think step by step and self reflect about mistakes on each step" can work
  on brand new models and fail in old or embedded models.


### Prompting Best patterns


A summary of 5 best prompting patterns or strategies extracted from OpenAI Prompt Engineering guide 
(<https://platform.openai.com/docs/guides/prompt-engineering>) <!-- [[PM.TODO.summarize]] -->

1. Write Clear Instructions:
  - Be specific: Clarity in instructions leads to more relevant outcomes.
  - Define the desired output length and complexity.
  - Demonstrate preferred formats.
  - Minimize ambiguity to enhance model accuracy.
2. Provide Reference Text:
  - Counteract potential fabrications with concrete reference materials.
  - Reference texts guide the model towards accurate and reliable answers.
3. Split Complex Tasks into Simpler Subtasks:
   - Break down tasks to reduce errors and improve manageability.
   - Consider tasks as workflows of simpler, interconnected steps.
4. Give the Model Time to "Think"
   - Allow the model to process and reason, similar to a human solving a complex problem.
   - Encourage a "chain of thought" approach for more accurate reasoning.
5. Use External Tools
   - Supplement the model's capabilities with specialized tools for specific tasks.
   - Leverage resources like text retrieval systems or code execution engines.
6. Test Changes Systematically
   - Measure improvements with a comprehensive testing approach
   - Ensure that modifications lead to overall performance enhancements.
