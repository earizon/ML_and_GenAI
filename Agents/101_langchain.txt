# LangChain

* Python framework Created in 2022 by Harrison Chase.

* Updated list of supported LLM models, key-value stores, graphs DBs, ... 
  <https://python.langchain.com/docs/integrations/llms/>
* Extensions allow for added functionality. Examples include:
  * LangSmith: Complements LangChain by providing robust debuggin, testing,
    and monitoring capabilities for LLM apps. Devs can quickly debug new
    chains by viewing detailed execution traces. Alternative prompts and LLMs
    can be evaluated againsts datasets to ensure quality and consistency. 
    Ussage analytics empowers data-drive decisions around optimizations.
  * LangChain templates: Comprehensive repo offering a suite of templates
    and central hub for resources.
  * LangServer: Help in deploy LangChain as REST APIs.
  * LangGraph: facilitates the development of complex LLMs with cyclic data
    flows and stateful multiactors scenarios.;

## Chains

* Chains orchestrates a sequences of operations such as input prompts,
  LLM invocations and output transformations.

  ```
  | # Example: Chain 1
  | from langchain              import LLMChain
  | from langchain_core.prompts import PromptTemplate

  | template01 = """ Question: {question}
  | Answer: Let's think step by step."""
  | prompt = PromptTemplate(template=template01,
  |   input_variables=["question"])

  | LLM_CHAIN = LLMChain ( 
  |   prompt=prompt, llm=LLM,
  |   verbose=True               # <·· 
  | )

  | LLM_CHAIN.run("What NFL team won the Super Bowl when Justin Beiber was born?")
  | # output will be similar to
  | # [1m> Entering new chain......
  | # Prompt after formatting: [[Question: What NFL ... born? Answer: ... by step
  | # [1m> Finished chain...
  | # Justin Beiber was born on March 1, 1994. The Super Bowl in 1994 was
  | # won by the San Francisco 49ers.
  ```

### LanChain Expression Language: LCEL

* Declarative (vs programatic) way to compose chains.
* builtin support for asynchronous processing, batching, streaming, 
  fallbacks, parallelism, and seamless integration with LangSmith tracing.

  ```
  | # Example: Chain using Mistral AI
  | from langchain_community.llms      import HuggingFaceEndpoint
  | from langchain_core.prompts        import PromptTemplate
  | from langchain_core.output_parsers import StrOutputParser
  | LLM = HuggingFaceEndpoint(
  |   repo_id="mistralai/Mistral-7B-Instruct-v0.2",
  |   max_length = 128, temperature = 0.5
  | )
  | template = """ Question: {question} Ansser: Let's think step by step """
  | # Force "Chain of Tought" (CoT) ···········─┴──────────────────────┘
  | prompt = PromptTemplate.from_template(template)
  | runnable = prompt | LLM | StrOutputParser()     # <·· CHAIN
  | #                 ^     ^
  | #                 └─────┴─ piping "a la UNIX".
  | summary = runnable.invoke({"question": "WHo won the FIFA World Cup in year 1994?"})
  | (The result will be similar to:
  |  FIFA World CUp ... is held every four years ...
  |  So, to find out which team won ..., we need to identify which World Cup
  |  tournament that was.
  |  First, we know that first FIFA World Cup was help in 1930,...
  |  ... To calculate ... we can subtract the year of the first World Cump...
  |  ... We'll round odnw to the nearest whole number ...
  |  ... Therefor, the answer is: Brazil won the FIFA World Cup in 1994.
  ```

## LangChain vs Parlant

* Parlant: <https://github.com/emcie-co/parlant>

  Parlant flips the script on AI agent development. Instead of hoping your LLM 
  will follow instructions, Parlant ensures it.

  ```
  | # Traditional approach: Cross your fingers 🤞
  | system_prompt = "You are a helpful assistant. Please follow these 47 rules..."
  | 
  | # Parlant approach: Ensured compliance
  | await agent.create_guideline(
  |     condition="Customer asks about refunds",
  |     action="Check order status first to see if eligible",
  |     tools=[check_order_status],
  | )
  ```

Parlant gives you all the structure you need to build customer-facing agents 
that behave exactly as your business requires:

* Journeys: Define clear customer journeys and how your agent should respond at 
  each step.
* Behavioral Guidelines: Easily craft agent behavior; Parlant will match the 
  relevant elements contextually.
* Tool Use: Attach external APIs, data fetchers, or backend services to 
  specific interaction events.
* Domain Adaptation: Teach your agent domain-specific terminology and craft 
  personalized responses.
* Canned Responses: Use response templates to eliminate hallucinations and 
  guarantee style consistency.
* Explainability: Understand why and when each guideline was matched and 
  followed.
