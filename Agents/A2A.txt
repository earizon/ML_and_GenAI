## Agent2Agent (A2A) protocol

* <https://github.com/google/A2A>

- A2A Core objective: Allow agents built on different frameworks and vendors to work together.

- A2A: Standard protocol to "expose" an Agent to another (probably remote) client Agent.
  A2A allows a (client) Agent to find another agent, let both agents expose each other their
  capabilities and negotiate how they will interact with users (via text, forms, or
  bidirectional audio/video), all while working securely together.

### A2A Specs

* <https://a2a-protocol.org/latest/specification/>

## A2A v1.0 concepts summary:

- A2A `Agent skills`: Define what the agent do. Attributes (defined in a2a.types) include:
  - id          : A unique identifier for the skill.
  - name        : A human-readable name.
  - description : A more detailed explanation of what the skill does.
  - tags        : Keywords for **categorization and discovery**. [[agents.discovery]]
  - examples    : Sample prompts or use cases.
  - input_modes/
    output_modes: Supported Media Types for inputs/outputs (e.g., "text/plain", "application/json").
  - security_requirements: Security schemes necessary for this skill.  [[security.*]]

- `Agent Card`  :<br/>
  - JSON document allowing other agents or clients to find out the 
    agent and its skills (capabilities).
  - Exposed through `.well-known/agent-card.json` HTTP endpoint path 
    relative to a given URL like `https://myagent.mycomp.com/`.
  - Other agents can access the Card directly (if the know the the 
    URL (hardcoded or injecting through configuration parameters)

   a well known URL or
   by querying an intermediate registry ("Find an agent with some given skill/s").

 It's like a digital business card for the agent.

Key attributes of an AgentCard (defined in a2a.types):

name, description, version: Basic identity information.
supported_interfaces: Ordered list of endpoints and protocols where the A2A service can be reached.
capabilities: Supported A2A features like streaming or extended_agent_card.
default_input_modes / default_output_modes: Default Media Types for the agent.
skills: A list of AgentSkill objects that the agent offers.
The helloworld example defines its Agent Card like this:


# Define a public-facing agent card that allows clients to discover your agent's capabilities.
public_agent_card = AgentCard(
    # Basic identity information of A2A server
    name='Hello World Agent',  # Identity
    description='Just a hello world agent',
    version='0.0.1',
    # Default Media Types for the agent's interactions
    default_input_modes=['text/plain'],  # Supported media types
    default_output_modes=['text/plain'],
    # Supported A2A features (like streaming or extended config)
    capabilities=AgentCapabilities(streaming=True, extended_agent_card=True),
    # Ordered list of endpoints and protocols where the service can be reached
    supported_interfaces=[
        AgentInterface(
            protocol_binding='JSONRPC',
            url='http://127.0.0.1:9999',
        )
    ],
    # The list of AgentSkill objects that this agent offers
    skills=[skill],
    # Optional attributes (omitted here for simplicity):
    # icon_url                         -> A URL to an icon representing the agent
)
This card tells us the agent is named "Hello World Agent", can be run at http://127.0.0.1:9999/, supports text interactions, and has the hello_world skill.

message SendMessageConfiguration { // Configuration of a send message request.
  repeated string accepted_output_modes // list of media types supported by client for response parts.

  TaskPushNotificationConfig task_push_notification_config

  optional int32 history_length // most recent msgs from task's history, unset => no limit, 0 => do not include any messages.
  // true   => Op. returns immediately after creating the task, even if processing is still in progress.
  // false  => operation MUST wait until the task reaches COMPLETED|FAILED|CANCELED|REJECTED terminal state (or interrupted)
  // ( INPUT_REQUIRED ,  AUTH_REQUIRED ) state before returning.
  bool return_immediatel

// Task: Core unit of action for A2A.
 It has a current status
// and when results are created for the task they are stored in the
// artifact. If there are multiple turns for a task, these are stored in
// history.
message Task
  string id // UUID (generated by the server)
  string context_id // UUID used to related tasks <-> messages.
  TaskStatus status // **CURRENT** status ==(state, message)
  repeated Artifact artifacts; // TIP: repeated => set of.
  repeated Message;
  google.protobuf.Struct metadata //  key/value object with custom metadata

enum TaskState
  TASK_STATE_UNSPECIFIED
  TASK_STATE_SUBMITTED           // task submitted and ACKNOWLEDGED.
  TASK_STATE_WORKING             // Actively being processed by the agent.
  TASK_STATE_COMPLETED           // OK terminal state.
  TASK_STATE_FAILED              // KO terminal state.
  TASK_STATE_CANCELED            // KO terminal state.
  TASK_STATE_INPUT_REQUIRED      // interrupted state.
  TASK_STATE_REJECTED            // KO terminal state. (rejected by agent)
    TASK_STATE_AUTH_REQUIRED     // interrupted state.

message TaskStatus { // A container for the status of a task
  TaskState state
  Message message // A message associated with the status.
  google.protobuf.Timestamp timestamp

// textual|image|video|...|structured data blob (JSON, ...).
message Part /* data container */ // string text | bytes raw | string url | google.protobuf.Value data
  + google.protobuf.Struct metadata // Metadata associated with this part.
  + filename       // optional
  string media_type

enum Role  // ROLE_UNSPECIFIED | ROLE_USER | ROLE_AGENT

// It can associated with a context and/or a task.
// For server messages:
// - context_id  must be provided,
// - task_id   must be provided if a task was created.
// For client messages:
// - context_id  optional
// - task_id     optional (if provided server will infer the context_id)
message Message { // unit of communication between client <-> server
  string message_id // UUID created by message creator
  string context_id // Optional. Links message with context.
  string task_id    // Optional. Links message with task.
  Role role
  repeated Part parts // message
  google.protobuf.Struct metadata // Optional.
  repeated string extensions // URIs of extensions present or contributed to this Message.
  repeated string reference_task_ids  // Optional. (additional context)

message Artifact { // == task outputs
  string artifact_id // UUID (unique within a task)
  string name // human readable
  string description // Optional.
  repeated Part parts // The content of the artifact. Must contain at least one part.
  google.protobuf.Struct metadata // Optional.
  repeated string extensions // URIs of extensions present or contributed to this Artifact.
