## HiPlot: High dimensional interactive visualization/ploting tool 

* <https://www.linkedin.com/posts/itay-gabbay_datascientist-python-datascientist-activity-7035911297799991296-RW-l>

- In many ML problems we usually need to handle high dimensional data.
- In classic ML applications it happen a lot, but with unstructured data
  (vision, nlp) this is almost always the case. [[101,data.unstructured]]

 Visualization of high-dimensional data is hard. When needed, datascientist
usually use dimensionality reduction techniques (e.g. PCA/SVD) that loses
some information of the data.

HiPlot is a python package that enables interactive visualizations to help
#datascientists discover correlations and patterns in high-dimensional data
using parallel plots and other graphical ways to represent information.

It can be used as a web server, within jupyter notebooks and even as a CLI
tool.
- Github: <https://lnkd.in/dVF6my7A>
- Docs: <https://lnkd.in/dE7sqT5b>

## SnakeMake: create reproducible and scalable data analyses

* <https://snakemake.readthedocs.io/en/stable/>

- Snakemake workflow management system is a tool to create
reproducible and scalable data analyses.
-  Workflows are described via human readable, Python based script.
   - essentially python + declarative code to define rules.
   - Rules describe how to create output files from input files.
- They can be seamlessly scaled to server, cluster, grid and cloud environments,
  without the need to modify the workflow definition. [[scalability.cluster]]
- Finally, Snakemake workflows can entail a description of required software,
   which will be automatically deployed to any execution environment.

Quick Example:


  ```
  | rule targets:
  |     input:
  |         "plots/myplot.pdf"
  | 
  | rule transform:
  |     input:
  |         "raw/{dataset}.csv"
  |     output:
  |         "transformed/{dataset}.csv"
  |     singularity:
  |         "docker://somecontainer:v1.0"
  |     shell:
  |         "somecommand {input} {output}"
  | 
  | rule aggregate_and_plot:
  |     input:
  |         expand("transformed/{dataset}.csv", dataset=[1, 2])
  |     output:
  |         "plots/myplot.pdf"
  |     conda:
  |         "envs/matplotlib.yaml"
  |     script:
  |         "scripts/plot.py"
  ```

## Scales Weak Supervision

(Overcome Labeled Dataset Problem)
* <https://www.infoq.com/news/2019/05/google-snorkel-drybell>

Google recognizes that the need for labeled data in machine learning (ML) is a 
significant bottleneck and recently adapted the open-source Snorkel framework 
to overcome the problem at scale.  ... Google documented the results ...
 [[PM.101,PM.bottleneck]]

- Snorkel uses software algorithms to create labels for training data
  instead of hand labeling data. [[data.gathering,data.labeling]]

- This technique is known as weak supervision. [[training.weak_supervision]]

- The algorithms can use any available organizational knowledge (graphs, 
  rules, statistics).

- Each algorithm can assign one or more labels or abstain from assigning a label.

- Then Snorkel automatically weights the algorithms based on an estimate of 
  their accuracy.

- It completes its work by creating a single probabilistic label for each
  data point.

- Google adapted Snorkel to create Snorkel Drybell with the expressed intent of 
  handling web-scale data.

...  Google did **NOT** enforce a strict context hierarchy for the data model
that represents training data, which was initially present in Snorkel. 

 Google also moved away from the usage of a database for storing data and
instead uses a distributed file system to share data. 
 Lastly, Google made the independent labeling functions separate executables
 that share data through the filesystem. 
 These changes allowed Google to scale the open source Snorkel project to
use large amounts of organizational knowledge at web scale 
[[persistence.file_system,scalability.architecture]]

... Google boosted its performance by an average of 52% 

P.S.: Snorkel is part of the Standford DAWN portfolio:
- "DAWN is a five-year research project to democratize AI by making
  it dramatically easier to build AI-powered applications. ...
